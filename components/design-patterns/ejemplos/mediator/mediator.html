Patrón Mediator

El patrón Mediator es un patrón de comportamiento que reduce el acoplamiento caótico entre objetos. El patrón restringe las comunicaciones directas entre los objetos y los fuerza a colaborar únicamente a través de un objeto mediador.

Propósito

Definir un objeto que encapsule cómo interactúan un conjunto de objetos.

Caso de Uso Real: Torre de Control Áereo o Chat Room

Contexto: Centenares de aviones volando cerca de un aeropuerto. Si cada avión tuviera que hablar con todos los demás para no chocar (A con B, A con C, B con C...), tendríamos un caos de comunicaciones (complejidad N*N).

El Problema: Demasiadas dependencias directas entre componentes hacen que el sistema sea imposible de mantener y testear. Cada avión necesitaría conocer a todos los demás.

La Solución (Mediator): Los aviones (Componentes) solo se comunican con la Torre de Control (Mediador).
Si un avión quiere aterrizar, se lo dice a la Torre. La Torre comprueba si la pista está libre y avisa a los otros aviones si es necesario. Las dependencias se reducen drásticamente (de N*N a N).

Análisis de la Implementación

  • Centralización del Control:

    El Mediador encapsula la lógica de interacción compleja. Los componentes individuales (User, Aircraft) se vuelven más simples y reutilizables porque no dependen de otros componentes concretos, solo del mediador.

  • Mediator vs Observer:

    En Observer, la comunicación es unidireccional (Subject -> Observers). En Mediator, suele ser multidireccional y el mediador contiene lógica de negocio para coordinar, no solo notificar.

Implementación


class ChatRoom {
    public static void showMessage(User user, String message) {
        System.out.println(new Date().toString() + " [" + user.getName() + "] : " + message);
    }
}

class User {
    private String name;
    public User(String name) { this.name = name; }
    public void sendMessage(String message) {
        ChatRoom.showMessage(this, message);
    }
}

Patrón Composite View

El patrón Composite View es un patrón de la arquitectura Java EE que permite construir una página completa ensamblando header, footer, menú y contenido, gestionados independientemente.

Propósito

Reutilización de componentes de interfaz y layouts consistentes (Tiles, SiteMesh).

Caso de Uso Real: Plantillas Maestras (Tiles / SiteMesh / Thymeleaf Layouts)

Contexto: Todas las páginas de tu web corporativa tienen el mismo Header, el mismo Footer y la misma Barra Lateral. Solo cambia el contenido central.

El Problema: Si copias y pegas el código del Header y Footer en cada una de las 200 páginas JSP, cuando quieras cambiar el logo tendrás un problema enorme. Los includes simples ayudan, pero escalar se vuelve difícil.

La Solución (Composite View): Creas una "Vista Compuesta" o Plantilla Maestra.
Esta plantilla define la estructura global y define "huecos" (slots) donde se inyectarán las sub-vistas concretas. El framework se encarga de ensamblar las piezas en tiempo de ejecución para generar la página final.

Análisis de la Implementación

  • Composición Atómica:

    Permite tratar partes de la página (como un Widget del Tiempo o un Carrito de Compra lateral) como componentes independientes reutilizables en múltiples vistas.

  • Consistencia Visual:

    Garantiza que todas las páginas sigan la misma estructura sin esfuerzo manual por parte del desarrollador, centralizando el diseño en un solo archivo de layout.

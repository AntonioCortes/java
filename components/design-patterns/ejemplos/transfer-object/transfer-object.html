Patrón Transfer Object (DTO)

El patrón Transfer Object (DTO) es un patrón de la arquitectura Java EE que transporta datos desde la capa de negocio a la de presentación en una sola llamada.

Propósito

Reducir el número de llamadas remotas transportando múltiples datos a la vez.

Caso de Uso Real: API REST o Comunicación entre Microservicios

Contexto: Un frontend en Angular necesita mostrar el perfil de usuario. En tu base de datos, la información del usuario está dispersa en 3 tablas: Users, Addresses, Roles.

El Problema: Si el frontend hace 3 llamadas REST (una para user, otra para address, otra para roles), la latencia de red hará que la carga se sienta lenta (Chattiness). Además, no quieres exponer tus entidades de base de datos JPA con contraseñas hash directamente al JSON.

La Solución (Transfer Object): El servidor hace las consultas locales rápidas, ensambla todos los datos necesarios en un único objeto POJO simple (UserProfileDTO) y lo envía en una sola respuesta.
El DTO actúa como un contenedor de datos optimizado para la transferencia por red.

Análisis de la Implementación

  • Reducción de Latencia:

    El objetivo principal es reducir el número de llamadas remotas. Es mejor traer un objeto grande de una vez que hacer 10 viajes pequeños a través de la red.

  • Desacoplamiento del Dominio:

    El DTO desacopla la estructura interna de tu base de datos (Entidades) de la estructura que expones al exterior (API Contract). Puedes cambiar tu modelo de datos sin romper el frontend, siempre que mantengas el DTO igual (o uses mappers).

Implementación


class UserDTO implements Serializable {
    public String username;
    public String email;
    // Getters, Setters, Constructor
}

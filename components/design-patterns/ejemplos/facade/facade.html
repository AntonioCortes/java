Patrón Facade

El patrón Facade es un patrón estructural que proporciona una interfaz simplificada a una biblioteca, un framework o cualquier otro grupo complejo de clases.

Propósito

Proveer una interfaz unificada y sencilla para un subsistema complejo.

Caso de Uso Real: Sistema de Cine en Casa (Smart Home)

Contexto: Tienes un sistema de entretenimiento complejo: Proyector, Amplificador, Reproductor DVD, Luces Inteligentes, Pantalla Motorizada. Ver una película implica 10 pasos: bajar pantalla, encender proyector, poner input HDMI, encender ampli, ajustar volumen, apagar luces, reproducir DVD.

El Problema: El cliente tiene que interactuar con demasiadas clases y conocer el orden exacto de los métodos. Es propenso a errores y difícil de usar.

La Solución (Facade): Creas una clase HomeTheaterFacade con un método simple watchMovie().
Esta clase se encarga internamente de orquestar todas las llamadas a los subsistemas en el orden correcto. El cliente solo presiona un botón (llama a un método).

Análisis de la Implementación

  • Interfaz Unificada:

    El Facade no añade funcionalidad nueva, solo expone una interfaz más simple para una funcionalidad existente. Actúa como un punto de entrada único.

  • Desacoplamiento débil:

    El Facade reduce el acoplamiento entre el cliente y el subsistema complejo, pero no impide que el cliente acceda a las clases de bajo nivel si realmente necesita un control fino. A diferencia de un Proxy o Adapter, el Facade suele crear una nueva interfaz más conveniente, no adaptar una existente.

Implementación


class VideoConverter {
    public File convert(String fileName, String format) {
        VideoFile file = new VideoFile(fileName);
        Codec sourceCodec = CodecFactory.extract(file);
        Codec destinationCodec = new MPEG4CompressionCodec();
        VideoFile buffer = BitrateReader.read(file, sourceCodec);
        VideoFile result = BitrateReader.convert(buffer, destinationCodec);
        return new File(result);
    }
}
// El cliente solo llama a VideoConverter.convert(...)

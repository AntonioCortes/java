Patrón Model View Controller (MVC)

El patrón Model View Controller (MVC) es un patrón de la arquitectura Java EE que arquitectura fundamental para aplicaciones interactivas.

Propósito

Separación de preocupaciones.

Caso de Uso Real: Arquitectura Web Moderna (Spring MVC)

Contexto: Estás construyendo una tienda online. Tienes datos del producto (Modelo), páginas HTML para mostrarlos (Vista) y lógica para procesar el carrito de compra (Controlador).

El Problema: Si mezclas código JDBC en tus JSPs (Vista), o generas HTML concatenando Strings dentro de tus Servlets (Controlador), creas un "código espagueti" que es imposible de testear y mantener. Un cambio en el diseño visual rompería la lógica de negocio.

La Solución (MVC): Separas estrictamente las responsabilidades:
1. Modelo (POJOs): Representa los datos y reglas de negocio. No sabe nada de HTML ni HTTP.
2. Vista (JSP/Thymeleaf/Angular): Solo se encarga de renderizar. No ejecuta lógica.
3. Controlador (Servlet/Spring Controller): Recibe la entrada del usuario, interactúa con el modelo y selecciona la vista adecuada.

Análisis de la Implementación

  • Desacoplamiento:

    Permite que diseñadores web trabajen en la Vista mientras desarrolladores backend trabajan en el Modelo y Controlador en paralelo sin pisarse.

  • Múltiples Vistas:

    El mismo Modelo (ej. lista de productos) puede ser reutilizado por múltiples Vistas (una página HTML, un PDF, un Excel, una respuesta JSON) sin cambiar una sola línea de lógica.

Implementación


// Model: Datos
class User { }

// View: HTML
// user-form.html

// Controller: Lógica
class UserController {
    public String saveUser(User user) {
        repo.save(user);
        return "success";
    }
}

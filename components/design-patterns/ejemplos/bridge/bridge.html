Patrón Bridge

El patrón Bridge es un patrón estructural que divide una clase grande o un conjunto de clases estrechamente relacionadas en dos jerarquías separadas (abstracción e implementación) que pueden desarrollarse independientemente.

Propósito

Desacoplar una abstracción de su implementación, de modo que ambas puedan variar independientemente.

Caso de Uso Real: Mando a Distancia Universal

Contexto: Quieres crear un sistema para controlar dispositivos (TV, Radio, Proyector) con distintos mandos a distancia. Cada dispositivo tiene sus propios comandos internos y cada mando puede tener funciones extra (Mute, Voz).

El Problema: Si usas herencia simple, acabarías con una explosión de clases: SonyTVRemote, SonyRadioRemote, SamsungTVRemote, etc. Si añades un nuevo tipo de mando o una nueva marca de TV, el número de clases crece exponencialmente (Producto cartesiano Abstracciones x Implementaciones).

La Solución (Bridge): Separas las jerarquías en dos:
1. Abstracción (Mando): Define las operaciones de alto nivel (encender, apagar).
2. Implementación (Dispositivo): Define las operaciones de bajo nivel (setVoltage, enableCircuit).
El mando guarda una referencia al dispositivo. Así, un solo mando genérico puede controlar cualquier dispositivo que implemente la interfaz común.

Análisis de la Implementación

  • Desacoplamiento Abstracción-Implementación:

    A diferencia de la herencia, donde la implementación está atada a la abstracción en tiempo de compilación, el Bridge permite cambiar la implementación en tiempo de ejecución (asignando un nuevo Device al RemoteControl). Esto favorece la composición sobre la herencia.

  • Extensibilidad Independiente:

    Puedes añadir nuevos mandos (ej. VoiceControlRemote) sin tocar las clases de dispositivos, y nuevos dispositivos (ej. SmartLight) sin tocar los mandos.

Implementación


// Implementor
interface Device {
    void turnOn();
    void turnOff();
    void setChannel(int channel);
}

// Abstraction
class RemoteControl {
    protected Device device;
    
    public RemoteControl(Device device) {
        this.device = device;
    }
    
    public void togglePower() {
        // lógica genérica
    }
}

// Refined Abstraction
class AdvancedRemoteControl extends RemoteControl {
    public AdvancedRemoteControl(Device device) {
        super(device);
    }
    
    public void mute() {
        device.setVolume(0);
    }
}

Estructura y capas de Spring

Un proyecto de SpringBoot está dividido en 3 capas principales, a saber:

  • Capa API
  • Capa de servicios
  • Capa de acceso a datos

La capa de API (Application Programming Interface) contiene todo lo relativo a la parte pública. En ella se encuentran las clases de DTO (Data Transfer Object), los controladores, los manejadores de excepciones, filtros, plantillas, etc

La capa de servicios contiene la lógica de negocio, así como toda la infraestructura necesaria para la comunicación de servicios con el resto de capas y con otros servicios ya sean de un mismo microservicio o de distintos.

La capa de acceso a datos contiene lo necesario para obtener y entregar datos persistentes de gestión privada. En esta capa se pueden econtrar los repositorios, las entidades, los modelos, las clases DAO( Data Access Object), etc.

Estructura de archivos

La documentación oficial de Spring plantea la posibilidad de organizar la estructura de archivos de la siguiente manera:

com +- example +- myapplication +- MyApplication.java | +- customer | +- Customer.java | +- CustomerController.java | +- CustomerService.java | +- CustomerRepository.java | +- order +- Order.java +- OrderController.java +- OrderService.java +- OrderRepository.java

Esta forma es muy similar a como Angular divide sus proyectos en componentes. No obstante según la experiencia de haber trabajado en distintos proyectos así como diversas fuentes no oficiales de origen indio, la forma de estructurar los archivos es agrupando controladores, servicios, entidades, etc en paquetes

com +- example +- myapplication +- MyApplication.java | +- Repository | +- CustomerRepository.java | +- FraudRepository.java | +- OrderRepository.java | +- Controller | +- CustomerController.java | +- FraudController.java | +- OrderController.java | +- Service +- CustomerService.java +- FraudService.java +- OrderService.java

En cualquier caso no hay un estandar fijado por spring, por lo que lo ideal es utilizar la manera que mejor se adapte al proyecto a efectos de mantenimiento, escalabilidad, legibilidad, etc

@SpringBootApplication

Todo proyecto de SpringBoot tiene una clase principal que contiene un método main el cual se utilizará para arrancar el proyecto. Esta clase se identifica con la anotación @SpringBootApplication

        
            @SpringBootApplication
            public class MyApplication {
            
                public static void main(String[] args) {
                    SpringApplication.run(MyApplication.class, args);
                }
            
            }

Ciclo de Vida

El ciclo de vida de Maven es un conjunto de fases predefinidas que se ejecutan en un orden específico para construir y empaquetar un proyecto. El ciclo de vida consta de tres fases principales:

  • La fase de validación, que valida que el proyecto está configurado correctamente y que todas las dependencias están disponibles.
  • La fase de compilación, que compila el código fuente del proyecto y genera los archivos de clase.
  • La fase de empaquetado, que toma los archivos compilados y los empaqueta en un formato distribuible, como un archivo JAR, WAR o ZIP.

Cada fase está compuesta por una serie de objetivos que se ejecutan en un orden predefinido. Por ejemplo, la fase de compilación puede incluir objetivos para compilar el código fuente, ejecutar pruebas y generar documentación. Los ciclos de vida están diseñados para que los desarrolladores puedan construir, probar y empaquetar sus proyectos de manera estandarizada y reproducible.

Comandos de Maven

Comando Descripción
mvn clean Cleans the project and removes all files generated by the previous build.
mvn compile Compiles source code of the project.
mvn test-compile Compiles the test source code.
mvn test Runs tests for the project.
mvn package Creates JAR or WAR file for the project to convert it into a distributable format.
mvn install Deploys the packaged JAR/ WAR file to the local repository.
mvn site generate the project documentation.
mvn validate validate the project’s POM and configuration.
mvn idea:idea generate project files for IntelliJ IDEA or Eclipse.
mvn release:perform Performs a release build.
mvn deploy Copies the packaged JAR/ WAR file to the remote repository after compiling, running tests and building the project.
mvn archetype:generate This command is used to generate a new project from an archetype, which is a template for a project. This command is typically used to create new projects based on a specific pattern or structure.
mvn dependency:tree This command is used to display the dependencies of the project in a tree format. This command is typically used to understand the dependencies of the project and troubleshoot any issues.

Operador ternario

El proposito de esta estructura es la asignación condicional.


//Sentencia del operador ternario
//Se pueden anidar operadores ternarios, aunque no es recomendable. 
//[CONDICION] ? [TRUE] : [FALSE]
nombre.equals("Jake") ? i++ : i--  


//Asignación condicional:
int a = 2;
int b = 5;
c = (a < b) ? a : b;
//c = 2

break: Permite salir del bucle del cual se le indique con etiquetas. En caso de no indicarse, sale del bucle donde se encuentre contenida dicha palabra reservada.

SWITCH

La expresión a evaluar puede ser de los tipos primitivos: byte, short, int y char; Y sus clases envoltorio. Además de String y clases numeradoras.

break: Permite finalizar la ejecución del switch. En caso de no indicar el break se ejecutan el resto de case siguiendo el orden en el que se encuentren.


switch (expression) { 
    case value1:
    // code
    break;

    case value2:
    // code
    break;

    case valueN :
    // code
    break;

    default:
    // Default code
}
